# 反馈闭环机制

## 目的
在每轮投递或面试后进行结构化迭代，避免无效重复修改。

## 每轮检查清单
1. 在 `application-tracker.csv` 记录本轮投递信息。
2. 记录反馈来源（JD 匹配、项目深度、叙事表达、沟通方式）。
3. 每轮最多只改 1-2 个变量：
- 简历摘要
- 作品集叙事
- 项目证据质量
- 面试回答结构
4. 以版本号方式迭代后再次投递。

## 版本命名规范
- 简历：`resume-v{n}`
- 作品集：`portfolio-v{n}`
- 项目文档：`projectA-v{n}`、`projectB-v{n}`

## 决策规则
同一拒绝原因连续出现 3 次时，优先做结构性调整（项目证据或岗位匹配），而不是只改措辞。
